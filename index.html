⓿dependencies!

Software with fewer dependencies, ideally ⓿, makes smart developers happy. If you don’t agree, this Web page won’t help you; it’s for people who already know.

Apps and libraries are different. It’s hard to write an app with ⓿ dependencies. But it’s always worthwhile trying to stay with libraries that are built-in to your programming language.

But if you’re writing a library, it’s worth trying for ⓿. Particularly one that has a chance of being widely used, or used by people with strict security requirements, or by people who need predictable performance, or all of the above.

That’s all I’m going to say, except that if you add the following to your README.md on GitHub, you’ll get a nice little badge that links here.
